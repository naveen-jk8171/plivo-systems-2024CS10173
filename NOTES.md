# NOTES.md

The sender and receiver implement a hybrid Forward Error Correction (FEC) and NACK-based Automatic Repeat reQuest (ARQ) protocol. The sender wraps each harness frame in a 165-byte wire packet (1-byte type + 4-byte seq + 160-byte payload). If the token bucket permits, it appends a consecutive parity packet (XOR of {i, i−1, i−2, i−3}) and an interleaved parity packet (XOR of {i, i−2, i−4, i−6}) to decorrelate burst losses across independent equations. The receiver forwards every new data packet to the player immediately (zero jitter buffer), then runs an iterative erasure solver that cascades recoveries across both parity types — an interleaved solve can unlock a consecutive solve and vice-versa. As a fallback, the receiver sends lightweight NACK packets every 5 ms listing missing sequences; the sender retransmits if tokens allow. A token bucket at 15,600 bytes/sec (≈1.95× raw) governs all non-primary transmissions, guaranteeing the 2× overhead cap is never breached while maximizing available bandwidth. Grade at `delay_ms=50` for Profile A and `delay_ms=110` for Profile B. The design breaks when losses arrive in bursts longer than 3 consecutive packets combined with near-maximum relay jitter, because no single parity equation retains enough known members to solve, and the ARQ round-trip (~160 ms through two hostile relay crossings) exceeds the remaining deadline budget.
